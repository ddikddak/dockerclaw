DockerClaw Agent API

REST API for AI agents to interact with DockerClaw boards and blocks.

Base URL

POST https://blendizvcpgdxiedjvqk.supabase.co/functions/v1/agent-execute

Authentication

All requests require an API key passed via the X-API-Key header.

curl -X POST $BASE_URL \
  -H "X-API-Key: dc_agent_YOUR_KEY_HERE" \
  -H "Content-Type: application/json" \
  -d '{"action": "list_boards"}'

API keys are created from the DockerClaw UI under Board Settings > Agent API. Keys are user-scoped and grant access to all boards owned by that user.

Request Format

Every request is a POST with a JSON body containing:

FieldTypeRequiredDescription
actionstringYesThe action to perform
board_idstring (UUID)For block actionsTarget board ID
paramsobjectNoAction-specific parameters

Actions

list_boards read

Returns all boards owned by the authenticated user.

{
  "action": "list_boards"
}

Response:

{
  "success": true,
  "action": "list_boards",
  "count": 2,
  "boards": [
    {
      "id": "d2fc9f7c-...",
      "name": "My Board",
      "created_at": "2026-02-27T15:32:31.201+00:00",
      "updated_at": "2026-03-02T13:10:12.135+00:00",
      "settings": { ... }
    }
  ]
}

list_blocks read

Returns all blocks on a board. Requires board_id.

{
  "action": "list_blocks",
  "board_id": "BOARD_UUID",
  "params": {
    "type": "doc",
    "include_deleted": false
  }
}
ParamTypeDefaultDescription
typestringallFilter by block type
include_deletedbooleanfalseInclude soft-deleted blocks

get_block read

Get a single block by ID.

{
  "action": "get_block",
  "board_id": "BOARD_UUID",
  "params": {
    "block_id": "BLOCK_UUID"
  }
}

create_block write

Create a new block on a board.

{
  "action": "create_block",
  "board_id": "BOARD_UUID",
  "params": {
    "type": "doc",
    "title": "Meeting Notes",
    "content": "# Agenda\n- Item 1\n- Item 2",
    "x": 200,
    "y": 150,
    "tags": ["meeting", "weekly"]
  }
}
ParamTypeRequiredDescription
typestringYesBlock type (see types below)
titlestringNoBlock title (maps to data.title or data.content for text/heading)
contentstringNoBlock content (maps to data.contentMarkdown for doc blocks)
x, ynumberNoPosition on canvas (default: 100, 100)
w, hnumberNoSize (defaults per type)
tagsstring[]NoTags (doc blocks only)
dataobjectNoRaw data object merged into block data

update_block write

Update an existing block. Fields are merged, not replaced.

{
  "action": "update_block",
  "board_id": "BOARD_UUID",
  "params": {
    "block_id": "BLOCK_UUID",
    "title": "Updated Title",
    "content": "New paragraph to append",
    "append": true,
    "x": 300,
    "y": 200
  }
}
ParamTypeDescription
block_idstringRequired. Block to update.
titlestringUpdate the title
contentstringUpdate content (contentMarkdown for docs)
appendbooleanIf true, append content instead of replacing (doc blocks)
x, y, w, h, znumberUpdate position/size/layer
lockedbooleanLock/unlock the block
tagsstring[]Replace tags (doc blocks)
dataobjectMerge into block data

Checklist operations

For checklist blocks, use checklist_action:

// Add item
{ "block_id": "...", "checklist_action": "add", "text": "Buy groceries" }

// Toggle item (by text match)
{ "block_id": "...", "checklist_action": "toggle", "text": "Buy groceries" }

// Check/uncheck by ID
{ "block_id": "...", "checklist_action": "check", "item_id": "ITEM_UUID" }

// Rename item
{ "block_id": "...", "checklist_action": "rename", "text": "Old name", "new_text": "New name" }

// Remove item
{ "block_id": "...", "checklist_action": "remove", "text": "Buy groceries" }

Valid checklist_action values: add, check, uncheck, toggle, remove, rename

delete_block delete

Soft-delete (move to trash) or permanently delete a block.

{
  "action": "delete_block",
  "board_id": "BOARD_UUID",
  "params": {
    "block_id": "BLOCK_UUID",
    "permanent": false
  }
}

clear_board write

Soft-delete all blocks on a board (moves to trash). Pass permanent: true to hard-delete.

{
  "action": "clear_board",
  "board_id": "BOARD_UUID",
  "params": {
    "permanent": false
  }
}
ParamTypeDefaultDescription
permanentbooleanfalseIf true, permanently deletes blocks. If false, soft-deletes (sets deleted_at).

Response:

{
  "success": true,
  "action": "clear_board",
  "deleted_count": 13,
  "permanent": false
}
Note: Soft-delete (default) syncs to connected clients via realtime. Use permanent: true only when you need irreversible deletion.

move_to_folder write

Move a block into a folder. The block data is embedded in the folder and the original block is removed from the canvas. No need to read or re-send block data — just pass IDs.

{
  "action": "move_to_folder",
  "board_id": "BOARD_UUID",
  "params": {
    "block_id": "BLOCK_UUID",
    "folder_id": "FOLDER_UUID"
  }
}
ParamTypeRequiredDescription
block_idstringYesID of the block to move
folder_idstringYesID of the target folder block

Response:

{
  "success": true,
  "action": "move_to_folder",
  "folder_id": "FOLDER_UUID",
  "moved_block_id": "BLOCK_UUID",
  "folder_item_id": "NEW_ITEM_UUID",
  "message": "Block moved into folder \"Sprint 14\""
}
Token-efficient: The server reads the block data and transforms it into a folder item automatically. AI agents only need to send two IDs.

move_from_folder write

Extract an item from a folder back onto the canvas as a standalone block. Pass the folder ID and item ID — the server handles creating the block and removing the item.

{
  "action": "move_from_folder",
  "board_id": "BOARD_UUID",
  "params": {
    "folder_id": "FOLDER_UUID",
    "item_id": "ITEM_UUID",
    "x": 500,
    "y": 200
  }
}
ParamTypeRequiredDescription
folder_idstringYesID of the folder containing the item
item_idstringYesID of the item inside the folder
x, ynumberNoPosition on canvas (default: next to folder)
w, hnumberNoSize (default: type-specific defaults)

Response:

{
  "success": true,
  "action": "move_from_folder",
  "folder_id": "FOLDER_UUID",
  "removed_item_id": "ITEM_UUID",
  "block": { ... },
  "message": "Item extracted from folder as new kanban block"
}
Tip: To list items in a folder, use get_block on the folder — items are in block.data.items with their id, type, title, and preview.

Permissions

API keys have granular permissions:

PermissionActions
readlist_boards, list_blocks, get_block
writecreate_block, update_block, clear_board, move_to_folder, move_from_folder
deletedelete_block

Error Responses

// 400 Bad Request
{ "error": "Missing required field: board_id" }

// 401 Unauthorized
{ "error": "Invalid or inactive API key" }

// 403 Forbidden
{ "error": "API key does not have write permission" }

// 404 Not Found
{ "error": "Block not found" }

// 409 Conflict
{ "error": "Block is deleted. Restore before updating." }

// 500 Internal Server Error
{ "error": "Failed to create block", "details": "..." }

# 1. List your boards
curl -X POST https://blendizvcpgdxiedjvqk.supabase.co/functions/v1/agent-execute \
  -H "X-API-Key: dc_agent_YOUR_KEY" \
  -H "Content-Type: application/json" \

  -H "Content-Type: application/json" \
  -d '{
    "action": "list_blocks",
    "board_id": "BOARD_UUID"
  }'

curl -X POST https://blendizvcpgdxiedjvqk.supabase.co/functions/v1/agent-execute \
  -H "X-API-Key: dc_agent_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "action": "create_block",
    "board_id": "BOARD_UUID",
    "params": {
      "type": "doc",
      "title": "Agent Report",
      "content": "# Summary\nGenerated by AI agent.",
      "x": 500,
      "y": 200
    }
  }'

# 4. Add a checklist item
curl -X POST https://blendizvcpgdxiedjvqk.supabase.co/functions/v1/agent-execute \
  -H "X-API-Key: dc_agent_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "action": "update_block",
    "board_id": "BOARD_UUID",

    }
  }'
Rate limits: API keys are subject to Supabase Edge Function limits. The last_used_at timestamp is updated on each request.
